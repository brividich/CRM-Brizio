1 — Applicazione
Parametri base dell'istanza Django. La SECRET_KEY deve essere una stringa casuale lunga almeno 50 caratteri.
Chiave crittografica Django. Non condividerla mai e non usare il valore di default.
Hostname/IP separati da virgola. Es: portale.cnovicrom.local,10.0.0.79
Attenzione: non abilitare DEBUG in produzione. Espone stack trace e configurazione interna.
2 — Sicurezza HTTPS
Imposta le opzioni SSL solo se il portale è raggiungibile tramite HTTPS (certificato valido). In rete interna senza TLS lascia tutto disabilitato.
Se il server ha un certificato SSL valido e risponde su HTTPS, abilita tutte e tre le opzioni sotto. Altrimenti lascia tutto a 0.

5 — Microsoft Graph / SharePoint
Credenziali Azure AD app registration per accesso a SharePoint (assenze, dipendenti, anomalie). Tutti i valori si trovano nel portale Azure.
Registra un'app in Azure Portal → App registrations con permessi: Sites.ReadWrite.All, Lists.ReadWrite.All.
ID sito SharePoint. Formato: nome.sharepoint.com,xxxxxxxx-...,xxxxxxxx-...

8 — Riepilogo & Download
Verifica la configurazione e scarica i file pronti per il server.
📄 django_app/.env
📄 config.ini
Posizionamento file:
• .env → django_app/.env sul server
• config.ini → cartella radice del progetto
• Non committare mai questi file nel repository Git!
Passi successivi:
1. Applicare gli script SQL in sql/ sul database SQL Server
2. python manage.py migrate --settings=config.settings.prod
3. python manage.py collectstatic --settings=config.settings.prod
4. Riavviare il server WSGI (gunicorn / IIS + wfastcgi)
